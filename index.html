WeHelp 第七屆申請

個人簡介

我是 Syrena 鐘晨瑄，畢業於高雄餐旅大學應用英語系。 過去曾在 Shopee做過產品優化實習生，也在 Asiayo擔任過市場開發實習生，主要是接觸電商和產品相關的工作。因為對網頁開發很有興趣，開始自學程式，後來也有機會在 Noonspace 擔任前端工程師七個月，實際參與公司專案。雖然我不是本科出身，但這一路跨過來，讓我累積了產品、使用者、工程不同角度的經驗。

申請動機

這兩年我主要靠 Udemy、freecodecamp、youtube 自學，也做過一些小專案。雖然有學到不少東西，但我發現自學常常會卡住，或是學到一半就換主題，沒有完整的架構。再加上缺乏同儕和導師的回饋，常常懷疑自己做出來的東西到底對不對？也因為技術更新很快，我感覺自己在基礎不穩的狀態下又需要去追新的技術，在這樣缺乏信心的情況下，很難有持續的成長。 我想參加 Wehelp，是因為儘管我能靠自己學習，但速度很慢、方向也不一定正確。我需要一個比較有系統的環境，能讓我把之前自學的東西整合起來，再加上實戰和團隊經驗，才有辦法真的走得更遠。 這次挑戰申請 Wehelp，我希望能夠：

  1. 穩固基礎，不再只靠零散的線上課程。
  2. 在專案中學習團隊合作。
  3. 儘管我很習慣自學，但希望能在對的方向努力。

學習經歷與作品

我主要透過 Udemy、YouTube 和 freeCodeCamp 學習網頁開發，包含 HTML、CSS、JavaScript，以及一些後端知識（Node.js）和 React。學習方式是「上課 → 練習 → 做小專案」。 過程中我也會用 AI 工具做 code review，但很多時候還是缺乏有人能幫我檢視、指導正確的做法。 目前我自己做過幾個作品，雖然這些專案幫助我累積了實作經驗，但我發現自學的知識常常零散、缺乏架構，而且作品都是自己一個人完成，缺乏團隊合作與專業檢視。這也是為什麼我想加入 Wehelp的原因，我需要一個系統化的環境和同儕互動，把這些基礎磨練成專業能力。

Modern E-commerce MVP

這是一個我用 HTML、CSS、JavaScript 完成的電商網站雛形。 功能包含商品頁面、基本互動效果，並練習了版面設計與排版。 雖然目前還沒有後端或金流，但這個專案幫助我把前端基礎練得更熟，也學會怎麼把作品部署到 Vercel，讓它能在網路上公開。

學習時間安排

目前我待業中，可以全心投入 Wehelp 的學習，一週安排大約 50 小時以上。學習安排大致上安排如下：

  • 週一：參加線上會議，掌握當週任務並拆解成每天的小目標。
  • 週二到週五：每天安排固定的學習與實作時間，專注在完成任務，並透過 Discord 討論與提問。
  • 進度回報：我會維持每日紀錄與回報，方便導師和自己追蹤進度。
  • 週末：檢討一週學習狀況，補齊落後進度，並整理成筆記或心得。

挫折與調適

上週我到一家外商廣告公司面試，後來收到拒絕信。雖然我事前有心理準備，但收到拒絕信當下心情還是不太美麗，一直反覆在想當天的過程是不是哪裡做錯了。 後來我給自己一點時間消化，去運動、去遛狗。當天晚上我就把過程寫下來，檢討自己在自我介紹和作品展示上的不足。也和朋友聊了一下，得到一些不同角度的回饋。 後來轉念想，其實這次經驗讓我更清楚知道公司看重什麼，也讓我在下一次面試時能更有準備。雖然被拒絕很不舒服，但我覺得這就是成長過程的一部分。

申請網頁的開發心得

在製作這份申請網頁時，我嘗試使用 CSS Grid 來排版。 它和我比較熟悉的 Flexbox 不太一樣，需要先規劃整個網格的結構，再把元素放進去。 一開始常常遇到格子跑位、對齊不齊的問題，不過在調整 row/column 的時候慢慢理解 Grid 的邏輯。 雖然還不算熟練，但我覺得Grid 幫助我更有系統地思考版面結構，也意識到不同工具各有適合的場景。

如何確認投入領域有回報 & 對AI的看法

我認為技術真的變得很快，尤其AI帶來的改變又是另外一個層面，但如果每個新的工具、技術都去追，最後可能什麼都學一半就放掉了。對我來說，比較重要的是先把基礎打穩，像 HTML、CSS、JavaScript 這些很基本一定要穩固的技能。 至於要不要投入新的領域，我大概會看三個面向：

  • 這個技術是不是很多公司在用，職缺上常看到。
  • 社群是不是活躍，文件、教學資源是不是多。
  • 它能不能解決實際問題，而不是只有流行但卻漏洞百出而已。

我現在主要專注在前端跟全端開發，因為可以馬上做出一個產品，也跟我的求職方向有關。像 AI、雲端這些我也有關注，但我不會急著跳去全學，最重要的是先把眼前用得到的技術學好，再慢慢延伸。

我認為 AI的確帶來很多方便，但它並不是萬能的。剛開始我也會擔心像我這樣的junior會不會被取代，不過後來的想法是：真正會被取代的，也許是不懂得使用AI工具的人。 對開發者來說，AI工具可以幫助我們更快完成重複性或瑣碎的工作，讓開發流程更有效率。所以我認為，反而是懂得結合 AI、保持學習能力的人，會在這個時代更有優勢。

從上次提出申請至今，多做了哪些努力

雖然上次申請失敗，不過我沒有放棄，還是持續透過 Udemy、YouTube 和freeCodeCamp 學習前端與全端開發，也做了一些自己的小專案。這些過程讓我學到很多，但也更明顯感受到自學的限制：學習比較零散，缺乏系統化的架構與專業回饋。 另外這段時間我也開始實際投履歷、參加面試，強迫自己去做原本會害怕的事情，才更清楚看到自己在專業能力和實戰經驗上的不足。這些經歷讓我確定，單靠自己摸索很難突破，因此想再次挑戰 Wehelp，期望在有結構的環境和同儕支持下，把這些努力轉化成真正能進入業界的能力。

想對 WeHelp 說的話

我想說聲謝謝，因為這樣的課程能用這麼親民的價格提供，對像我這樣非本科、靠自學轉職的人來說真的很有幫助。即使上次沒有錄取，我還是覺得這是一個很珍貴的機會，所以才會想再挑戰一次。希望這次有機會能跟著大家一起學習、一起成長。
